# ЭТАП 1: РЕЗЮМЕ ПРОЕКТА

**Версия:** 1.0  
**Дата:** 2025  
**Статус:** Проектирование

---

## Краткое описание

Первый этап проекта автоматизации анализа аварийных сигналов представляет собой алгоритмическую систему без использования машинного обучения. Система автоматически анализирует тренды параметров оборудования, проверяет соответствие уставок фактическим параметрам, выявляет ложные сигналы и генерирует рекомендации по устранению проблем.

---

## Цели этапа

1. **Автоматизация анализа** - снижение времени на анализ аварийных сигналов
2. **Выявление проблем** - автоматическое обнаружение несоответствий уставок и проблем с оборудованием
3. **Генерация рекомендаций** - автоматическое формирование рекомендаций по устранению проблем
4. **Снижение ложных тревог** - выявление и отключение ложных сигналов
5. **Улучшение процессов** - оптимизация работы персонала

---

## Ключевые возможности

### 1. Автоматический анализ трендов

- Анализ исторических данных за 2 месяца
- Расчет статистических показателей (среднее, минимум, максимум, медиана, стандартное отклонение)
- Сравнение текущего и предыдущего периодов
- Выявление трендов и аномалий

### 2. Проверка соответствия уставок

- Сравнение фактических параметров с текущими уставками
- Проверка соответствия нормативным требованиям
- Проверка соответствия паспортным данным оборудования
- Расчет рекомендуемых значений уставок

### 3. Выявление ложных сигналов

- Определение сигналов от неработающего оборудования
- Выявление сигналов при нормальных технологических процессах
- Определение сигналов во время плановых работ

### 4. Генерация рекомендаций

- Изменение уставок
- Замена оборудования
- Отключение сигналов
- Исключение сигнализации при определенных условиях
- Проведение ЭПБ
- Разработка мероприятий

### 5. Интерфейсы для различных ролей

- Дашборд для диспетчеров ЕДС
- Интерфейс анализа объектов
- Управление уставками
- Управление рекомендациями
- Отчеты

## Архитектура системы

### Компоненты

1. **Слой интеграции:**
   - API Gateway
   - Data Sync Service
   - File Parser Service

2. **Слой бизнес-логики:**
   - Trend Analyzer
   - Threshold Validator
   - Equipment Status Checker
   - Alarm Classifier
   - Recommendation Generator
   - Notification Service

3. **Слой данных:**
   - Time Series Database (TimescaleDB)
   - Metadata Database (PostgreSQL)
   - Recommendations Database (PostgreSQL)
   - Cache Layer (Redis)

4. **Слой интерфейсов:**
   - Web Application (React)
   - Mobile Application (React Native)
   - REST API
   - Report Generator

### Интеграции

- АСДКУ - данные параметров в реальном времени
- ИС «МЕГА» - исторические данные
- ИС «Проток» - исторические данные
- ЕССД - список аварийных сигналов
- КИС «Армитс» - объемы перекачки, прогнозы
- 1С ТОиР - метаданные оборудования
- СЭД - перечни блокировок и сигнализаций

---

## Алгоритмы анализа

### Основные алгоритмы

1. **Анализ давления в емкости:**
   - Загрузка данных давления и уровня
   - Расчет статистики
   - Проверка состояния емкости
   - Проверка допустимых параметров

2. **Анализ давления на приеме насоса:**
   - Анализ при работе насоса
   - Корреляция с давлением в емкости
   - Корреляция с объемами перекачки
   - Прогноз на основе объемов

3. **Анализ давления на выкиде насоса:**
   - Проверка условий сработки сигнала
   - Исключение сигналов при остановленном насосе
   - Проверка соответствия допустимым параметрам

4. **Выявление ложных сигналов:**
   - Проверка состояния оборудования
   - Проверка условий сработки
   - Определение причин ложных сигналов

5. **Проверка соответствия уставок:**
   - Сравнение с фактическими параметрами
   - Проверка нормативных требований
   - Расчет рекомендуемых значений

## Технический стек

### Backend

- **Язык:** Python 3.11+ или Node.js 20+
- **Фреймворк:** FastAPI (Python) или Express.js (Node.js)
- **Базы данных:** PostgreSQL 15+, TimescaleDB, Redis
- **Обработка данных:** Pandas, NumPy, SciPy
- **Парсинг:** PyPDF2, python-docx, openpyxl

### Frontend

- **Фреймворк:** React 18+, TypeScript 5+
- **UI библиотеки:** Material-UI v5
- **Графики:** Chart.js или Recharts
- **Состояние:** React Query, Zustand
- **Формы:** React Hook Form, Zod

### Инфраструктура

- **Контейнеризация:** Docker, Docker Compose
- **Веб-сервер:** Nginx
- **Мониторинг:** Prometheus, Grafana, ELK Stack
- **CI/CD:** GitLab CI/CD или GitHub Actions

## План реализации

### Этапы разработки

1. **Подготовка и проектирование** (2-3 недели)
2. **Разработка базовой инфраструктуры** (3-4 недели)
3. **Разработка алгоритмов анализа** (4-5 недель)
4. **Разработка веб-интерфейса** (4-5 недель)
5. **Интеграция и тестирование** (3-4 недели)

**Общая длительность разработки:** 16-21 неделя (4-5 месяцев)

### Пилотное внедрение

- **Объекты:** 5-10 объектов ДНС, ГЗНУ из одного НГДУ
- **Длительность:** 8 недель
- **Цель:** Проверка работоспособности на реальных данных

### Развертывание

1. **Фаза 1:** Пилотные объекты (2 недели)
2. **Фаза 2:** Все объекты ДНС, ГЗНУ (4-6 недель)
3. **Фаза 3:** Другие типы объектов (4-6 недель)

**Общая длительность развертывания:** 10-14 недель (2.5-3.5 месяца)
